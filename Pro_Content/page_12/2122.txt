<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=2122&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">Japan Plotter Driver</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 619</td><td width="10px"></td><td><b>Accepted:</b> 273</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">The Japan company you are working for produce plotter devices that can draw nice pictures. To support customers who do not posses the special hardware, you were asked to write an emulation driver that simulates the work of the plotter and prints the picture on a computer screen. <br><br>The plotter is driven with a simple language consisting of several drawing commands: <br><ul><li>POINT x y -- makes a little circle at the given coordinates. <br><li>TEXT x y txt -- displays a line of text at the given coordinates. <br><li>LINE x1 y1 x2 y2 -- draws a line between the specified points. <br><li>CLEAR x1 y1 x2 y2 -- erases the given rectangle. <br><li>PRINT -- prints an output page and terminates the current job. </ul><br>The emulation driver uses few ASCII characters to represent the picture, one character being a basic unit of the coordinate system. The top-left character has coordinates (1,1). The X-axis aims to the right, the Y-axis goes down. <br><br>The particular commands are emulated as follows: <br><ul><li>POINT: The driver puts a lowercase letter "o" at the given coordinates. <br><li>TEXT: Shows a single line of text, the first character is positioned at the given coordinates and the text always goes right. <br><li>LINE: Simulates a straight line between two points. The line is formed by one of the following characters: dash ("-"), pipe ("|"), slash ("/"), or backslash ("\"), according to its direction. <br><li>CLEAR: The driver fills the appropriate rectangular area with spaces, including the bounding rows and columns. <br><li>PRINT: This command causes the driver to print the picture surrounded with a nice frame made of plus ("+"), minus ("-"), and pipe ("|") characters. </ul><br>If more objects should be drawn across a single character, the following rules apply: <br><ul><li>If the same character is drawn several times, it is used without a change. <br><li>If only pipe and minus characters are involved, they result in the plus sign ("+"). <br><li>If only slashes and backslashes are involved, the result is the lowercase letter "x". <br><li>Otherwise, the asterisk ("*") is displayed. </ul><br>Before a script is given to the driver, it is checked by a special preprocessor, which rejects all invalid commands. Therefore, you may assume that all coordinates are within the range of the page. Also, with the LINE command, the two points are always different and the line is strictly either vertical, horizontal, or at the angle of 45<sup>o</sup> to the axes. There is no assumption on the relative position of the points used with the LINE and CLEAR commands. The text in the TEXT command is always composed only of uppercase letter and digits. <br><br></div><p class="pst">Input</p><div class="ptx" lang="en-US">The input consists of several scripts. Each script begins with a line containing two integers X and Y, separated with space, 1 <= X, Y <= 75. These numbers specify the dimensions of the page. Every other line contains exactly one of the above commands. The commands are always uppercase, command arguments are separated with one space. <br><br>The PRINT command is always the last command of the script. After the PRINT command, a new script begins. The input is terminated with two zeros, which are not considered to be a script. </div><p class="pst">Output</p><div class="ptx" lang="en-US">For each script, output the emulated picture, created as specified above. After the picture, print one blank line.</div><p class="pst">Sample Input</p><pre class="sio">20 10
