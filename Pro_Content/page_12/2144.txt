<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=2144&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">Leaky Cryptography</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 5000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 447</td><td width="10px"></td><td><b>Accepted:</b> 328</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">The ACM ICPC judges are very careful about not leaking their problems, and all communi-cations are encrypted. However, one does sometimes make mistakes, like using too weak an encryption scheme. Here is an example of that.<br>The encryption chosen was very simple: encrypt each chunk of the input by flipping some bits according to a shared key. To provide reasonable security, the size of both chunk and key is 32 bits.<br>That is, suppose the input was a sequence of m 32-bit integers.<br>N1 N2 N3 ... Nm<br>After encoding with the key K it becomes the following sequence of m 32-bit integers.<br>(N1∧K) (N2∧K) (N3∧K) ... (Nm∧K)<br>where (a∧b) is the bitwise exclusive or of a and b.<br>Exclusive or is the logical operator which is 1 when only one of its operands is 1, and 0 otherwise.<br>Here is its definition for 1-bit integers.<br><pre>0 ♁ 0 = 0     0 ♁ 1 = 1<br>1 ♁ 0 = 1     1 ♁ 1 = 0</pre><br>As you can see, it is identical to addition modulo 2. For two 32-bit integers a and b, their bitwise exclusive or a ∧ b is defined as follows, using their binary representations, composed of 0's and 1's.<br>a ∧ b = a<sub>31</sub>...a<sub>1</sub>a<sub>0</sub>∧b<sub>31</sub>...b<sub>1</sub>b<sub>0</sub> = c<sub>31</sub>...c<sub>1</sub>c<sub>0</sub><br>where <br>c<sub>i</sub> = a<sub>i</sub>♁b<sub>i</sub> (i = 0, 1, ... , 31).<br>For instance, using binary notation, 11010110 ∧ 01010101 = 10100011, or using hexadecimal, d6 ∧ 55 = a3.<br>Since this kind of encryption is notoriously weak to statistical attacks, the message has to be compressed in advance, so that it has no statistical regularity. We suppose that N1 N2 . . .Nm is already in compressed form.<br>However, the trouble is that the compression algorithm itself introduces some form of regularity: after every 8 integers of compressed data, it inserts a checksum, the sum of these integers. That is, in the above input, N9 =  N1 +...+ N8, where additions are modulo 2<sup>32</sup>.<br>Luckily, you could intercept a communication between the judges. Maybe it contains a problem for the finals!<br>As you are very clever, you have certainly seen that you can easily find the lowest bit of the key, denoted by K0. On the one hand, if K0 = 1, then after encoding, the lowest bit of Σ<sub>1<=i<=8</sub>Ni∧K is unchanged, as K0 is added an even number of times, but the lowest bit of N9∧K is changed, so they shall differ. On the other hand, if K0 = 0, then after encoding, the lowest bit of Σ<sub>1<=i<=8</sub>Ni∧K shall still be identical to the lowest bit of N9∧K, as they do not change. For instance, if the lowest bits after encoding are 1 1 1 1 1 1 1 1 1 then K0 must be 1, but if<br>they are 1 1 1 1 1 1 1 0 1 then K0 must be 0.<br>So far, so good. Can you do better?<br>You should find the key used for encoding.</div><p class="pst">Input</p><div class="ptx" lang="en-US">The input starts with a line containing only a positive integer S, indicating the number of datasets in the input. S is no more than 1000.<br>It is followed by S datasets. Each dataset is composed of nine 32-bit integers corresponding to the first nine chunks of a communication. They are written in hexadecimal notation, using digits `0' to `9' and lowercase letters `a' to `f', and with no leading zeros. They are separated by a space or a newline. Each dataset is ended by a newline.</div><p class="pst">Output</p><div class="ptx" lang="en-US">For each dataset you should output the key used for encoding. Each key shall appear alone on its line, and be written in hexadecimal notation, using digits `0' to `9' and lowercase letters `a' to `f', and with no leading zeros.</div><p class="pst">Sample Input</p><pre class="sio">8
