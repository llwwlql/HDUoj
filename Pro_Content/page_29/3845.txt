<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td colspan="3"><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 368</td><td width="10px"></td><td><b>Accepted:</b> 148</td><td width="10px"></td><td style="font-weight:bold; color:red;">Special Judge</td></tr></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Fractals are really cool mathematical objects. They have a lot of interesting properties, often including:<br>1. fine structure at arbitrarily small scales;<br>2. self-similarity, i.e., magnified it looks like a copy of itself;<br>3. a simple, recursive definition.<br>Approximate fractals are found a lot in nature, for example, in structures such as clouds, snow flakes, mountain ranges, and river networks.<br><center><img src=images/3845_1.png></center><br>In this problem, we consider fractals generated by the following algorithm: we start with a polyline, i.e., a set of connected line segments. This is what we call a fractal of depth one (see leftmost picture). To obtain a fractal of depth two, we replace each line segment with a scaled and rotated version of the original polyline (see middle picture). By repetitively replacing the line segments with the polyline, we obtain fractals of arbitrary depth and very fine structures arise. The rightmost picture shows a fractal of depth three.<br>The complexity of an approximate fractal increases quickly as its depth increases. We want to know where we end up after traversing a certain fraction of its length.</div><p class="pst">Input</p><div class="ptx" lang="en-US">The input starts with a single number c (1 <= c <= 200) on one line, the number of test cases. Then each test case starts with one line with n (3 <= n <= 100), the number of points of the polyline. Then follow n lines with on the ith line two integers xi and yi ( -1 000 <= xi, yi <= 1 000), the consecutive points of the polyline. Next follows one line with an integer d (1 <= d <= 10), the depth of the fractal. Finally, there is one line with a floating point number f (0 <= f <= 1), the fraction of the length thatis traversed.<br>The length of each line segment of the polyline is smaller than the distance between the first point (x1, y1) and the last point (xn, yn) of the polyline. The length of the complete polyline is smaller than twice this distance.</div><p class="pst">Output</p><div class="ptx" lang="en-US">Per test case, the output contains one line with the coordinate where we end up. Format it as (x,y), with two floating point numbers x and y. The absolute error in both coordinates should be smaller than 10<sup>-6</sup>.</div><p class="pst">Sample Input</p><pre class="sio">1
4
-2 -2
0 0
0 2
2 2
3
0.75</pre><p class="pst">Sample Output</p><pre class="sio">(0.4267766953,2)</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=2009+ACM+North+Western+European+Regional+Contest">2009 ACM North Western European Regional Contest</a></div></td></tr></table>
