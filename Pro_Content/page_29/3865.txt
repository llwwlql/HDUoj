<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=3865&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">Database</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 5000MS</td><td width="10px"></td><td colspan="3"><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 1450</td><td width="10px"></td><td><b>Accepted:</b> 277</td><td width="10px"></td><td style="font-weight:bold; color:red;">Special Judge</td></tr></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Peter studies the theory of relational databases. Table in the relational database consists of values that are arranged in rows and columns.<br>There are different normal forms that database may adhere to. Normal forms are designed to minimize the redundancy of data in the database. For example, a database table for a library might have a row for each book and columns for book name, book author, and author's email.<br>If the same author wrote several books, then this representation is clearly redundant. To formally define this kind of redundancy Peter has introduced his own normal form. A table is in Peter's Normal Form (PNF) if and only if there is no pair of rows and a pair of columns such that the values in the corresponding columns are the same for both rows.<br><center><table border="1"><tr><td>How to compete in ACM ICPC</td><td>Peter</td><td>peter@neerc.ifmo.ru</td></tr><tr><td>How to win ACM ICPC</td><td>Michael</td><td>michael@neerc.ifmo.ru</td></tr><tr><td>Notes from ACM ICPC champion</td><td>Michael</td><td>michael@neerc.ifmo.ru</td></tr></table></center><br>The above table is clearly not in PNF, since values for 2rd and 3rd columns repeat in 2nd and 3rd rows.<br>However, if we introduce unique author identifier and split this table into two tables - one containing book name and author id, and the other containing book id, author name, and author email, then both resulting tables will be in PNF.<br><center><table border="1"><tr><td>How to compete in ACM ICPC</td><td>1</td></tr><tr><td>How to win ACM ICPC</td><td>2</td></tr><tr><td>Notes from ACM ICPC champion</td><td>2</td></tr></table></center><br><br><center><table border="1"><tr><td>1</td><td>Peter</td><td>peter@neerc.ifmo.ru</td></tr><tr><td>2</td><td>Michael</td><td>michael@neerc.ifmo.ru</td></tr></table></center><br>Given a table your task is to figure out whether it is in PNF or not.<br></div><p class="pst">Input</p><div class="ptx" lang="en-US">The first line of the input file contains two integer numbers n and m (1 &lt;= n &lt;= 10 000, 1 &lt;= m &lt;= 10), the number of rows and columns in the table. The following n lines contain table rows. Each row has m column values separated by commas. Column values consist of ASCII characters from space (ASCII code 32) to tilde (ASCII code 126) with the exception of comma (ASCII code 44). Values are not empty and have no leading and trailing spaces. Each row has at most 80 characters (including separating commas).</div><p class="pst">Output</p><div class="ptx" lang="en-US">If the table is in PNF write to the output file a single word "YES" (without quotes). If the table is not in PNF, then write three lines. On the first line write a single word "NO" (without quotes). On the second line write two integer row numbers r1 and r2 (1 &lt;= r1, r2 &lt;= n, r1 <img src="formula?tex=%5Cne" alt="\ne" align="absmiddle"> r2), on the third line write two integer column numbers c1 and c2 (1 &lt;= c1, c2 &lt;= m, c1 <img src="formula?tex=%5Cne" alt="\ne" align="absmiddle"> c2), so that values in columns c1 and c2 are the same in rows r1 and r2.</div><p class="pst">Sample Input</p><pre class="sio">Sample Input #1:
3 3
How to compete in ACM ICPC,Peter,peter@neerc.ifmo.ru
How to win ACM ICPC,Michael,michael@neerc.ifmo.ru
Notes from ACM ICPC champion,Michael,michael@neerc.ifmo.ru

Sample Input #2:
2 3
1,Peter,peter@neerc.ifmo.ru
2,Michael,michael@neerc.ifmo.ru</pre><p class="pst">Sample Output</p><pre class="sio">Sample Output #1:
NO
2 3
2 3

Sample Output #2:
YES</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Northeastern+Europe+2009">Northeastern Europe 2009</a></div></td></tr></table>
