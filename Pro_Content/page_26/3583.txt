<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 957</td><td width="10px"></td><td><b>Accepted:</b> 138</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><p>frkstyc has always wanted POJ to have a multiligual user interface. He has considered several schemes to implement the functionalities. Unfortunately, he has been occupied during the recent months, which prevents him from working on the development of POJ. Still, he would like to share some of his ideas in the hope that someone could finish the work for him.</p><p>One challenge in implementing a multilingual user interface is the manipulation of text messages. As you can easily identify, POJ has been developed with English as the primary language in its user interface. Hundreds of text messages that appear in web pages scatter in tens of source files as string resources. Isolating these string resources in the source files to faciliate the use of a unified programming interface for multilingualism support is itself a tedious and time-comsuming job and frequently triggers dismay in frkstyc. Besides, there are still some subtleties concerning differences between languages.</p><p style="text-align: left;">One particular such tricky aspect in which frkstyc is interested is word order. Consider the caption of the table in the page of Login Log. In English, it shall read “Last <tt>num</tt> login attempts by <tt>user</tt>”, where <tt>user</tt> appears after <tt>num</tt>. In Chinese, in contrast, it shall read “<tt>user</tt>的最后<tt>num</tt>次登录尝试”, where <tt>user</tt> appears before <tt>num</tt>. Such difference causes complexity. You may argue that the caption in English can be changed to “<tt>user</tt>’s last <tt>num</tt> login attempts” to avoid the trouble with Chinese. However, in the case of Spanish, it may not be that easy to reorder the words in “Los últimos <tt>num</tt> intentos de <tt>user</tt> por acceder” as in English.</p><p style="text-align: left;">frkstyc’s proposed solution is rather simplistic. He uses format strings with numbered placeholders. Take the aforementioned text messages for example. frkstyc creates the following three format strings</p><ul><li>Last {1} login attempts by {2}</li><li>{2}的最后{1}次登录尝试</li><li>Los últimos {1} intentos de {2} por acceder</li></ul><p>where {1} and {2} are the numbered placeholders. By supplying a format string and the array of arguments [<tt>num</tt>, <tt>user</tt>] to a formatting function which replaces {1} with <tt>num</tt> and {2} with <tt>user</tt>, frkstyc always gets desired result regardless of the choice of language. Furthermore, the backslash is designated as the escape character. An unescaped backslash escapes the immediately following character, which shall be directly copied to the output and not further processed. A placeholder is recognized if none of its comprised characters is escaped and the number immediately surrounded by braces is a valid 1-based index into the array of arguments. A recognized placeholder shall be replaced in the output by array element indexed by the number in braces.</p><p>frkstyc recognizes writing the formatting function as the only fun part in implementing the envisioned multilingualism support. He delightfully invites you to share such joy by also writing the function.</p></div><p class="pst">Input</p><div class="ptx" lang="en-US"><p>The input consists of a single test case. The test case contains at most 1001 lines, each being a string of at most 80 printable ASCII characters. The first string is the format string. The rest comprises the array of arguments.</p></div><p class="pst">Output</p><div class="ptx" lang="en-US">The output shall contain only the formatted string.</div><p class="pst">Sample Input</p><pre class="sio">Last {1} login attempts by {2}
20
frkstyc</pre><p class="pst">Sample Output</p><pre class="sio">Last 20 login attempts by frkstyc</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=POJ+Founder+Monthly+Contest+%E2%80%93+2008.04.13">POJ Founder Monthly Contest – 2008.04.13</a>, frkstyc</div></td></tr></table>
