<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 10000K</td></tr><tr><td><b>Total Submissions:</b> 2687</td><td width="10px"></td><td><b>Accepted:</b> 943</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Tester Program<br><br>For this contest, we first designed the following problem (note that you do not have to solve it!):<br><br><h1>Another Wall in the Maze</h1><br>In ACM/ICPC contests, you'll often see questions such as "find the shortest path out of this maze." Let's turn this on its head and ask "given a path, find a maze for which the given path is the shortest path." Our paths will run vertically and horizontally between the regularly spaced points of a rectangular grid. The problem is to compute a set of unit-length baffles (walls) separating grid points that forces the given path to be the unique shortest path from its starting point to the end point. To make things more interesting, we will require that there should be no redundant walls constructed in the sense that it should not be possible to remove any wall and still have the given path as the unique shortest path. In the following figure for example, consider the path through the 8 ? 5 grid on the left maze of the top row. The wall placements in the two mazes to its right (top row) make that path unique. The two mazes on the lower row are faulty.<br>The path is not unique in the one on the left, and there are some redundant walls on the right.<br><img src="images/1024/maze.gif"><br><h2>Input (of the original problem)</h2><br>The first line of the input file contains a single integer t (1 ≤ t ≤ 10), the number of test cases, followed by the input data for each test case. The first line of each test case consists of two integers W and H (1 ≤ W, H ≤ 100) giving the width and height of the grid respectively. The second line of the test case contains a path. The path always starts in the lowerleft corner, (0, 0). It is specified as a string of U (up), D (down), L (left), and R (right) characters (with no embedded white space). You may assume that the path remains within the bounds of the maze and does not intersect itself. It may end anywhere in the maze (i.e., not necessarily in a corner or against a wall).<br><h2>Output (of the original problem)</h2><br>First line of the output for the i-th test case (starting from one) should contain an integer M, the number of walls used in the solution. Following the first line, there are M lines each containing a wall specification in the form of four consecutive integers corresponding to two pairs of (x, y) coordinates specifying adjacent grid points separated by the wall (0 ≤ x < W and 0 ≤ y < H). Note that the possible output is not unique. There should no blank line in the output.<br><h2>Sample Input (of the original problem)</h2><br>2<br>8 5<br>RRRUULLURRRRDDRRUUU<br>4 3<br>RRRUU<br><h2>Sample Output (of the original problem)</h2><br>19<br>0 0 0 1<br>1 0 1 1<br>2 0 2 1<br>2 1 3 1<br>3 0 4 0<br>3 1 4 1<br>3 2 4 2<br>3 2 3 3<br>2 2 2 3<br>4 2 4 3<br>0 3 0 4<br>1 3 1 4<br>2 3 2 4<br>3 3 3 4<br>4 3 4 4<br>5 3 5 4<br>5 3 6 3<br>5 2 6 2<br>6 1 6 2<br>2<br>2 2 3 2<br>2 2 2 1<br>This is the end of the original problem statement! Being lazy, we did not want to spend time to write a tester program for this problem, and decided to have you write this for us!<br>Write a program that receives both input and output as one input test case, and write as output CORRECT or INCORRECT to indicate whether or not the output is correct.</div><p class="pst">Input</p><div class="ptx" lang="en-US">You read both input and output of the original problem from the standard input;it has each output just after each case's input of the original problem.<br>Note that the output of original does not have formatting problems, i.e.,<br>  The number of lines in the output file is correct and is as supposed to be.<br>  There are no leading or trailing white space characters in output lines.<br>  Wall specifications are correct, meaning that the four numbers correctly specify a possible wall within the boundary of the maze.</div><p class="pst">Output</p><div class="ptx" lang="en-US">Your program should write a single line for each test case of the input containing a single word CORRECT or INCORRECT, indicating the original problem has correctly produced the output for that test case or not.</div><p class="pst">Sample Input</p><pre class="sio">2
