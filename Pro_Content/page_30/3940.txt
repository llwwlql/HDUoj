<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=3940&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">Grey Area</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td colspan="3"><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 285</td><td width="10px"></td><td><b>Accepted:</b> 142</td><td width="10px"></td><td style="font-weight:bold; color:red;">Special Judge</td></tr></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Dr. Grey is a data analyst, who visualizes various aspects of data received from all over the world everyday. He is extremely good at sophisticated visualization tools, but yet his favorite is a simple self-made histogram generator.<br>Figure 1 is an example of histogram automatically produced by his histogram generator. A histogram is a visual display of frequencies of value occurrences as bars. In this example, values in the interval 0–9 occur five times, those in the interval 10–19 occur three times, and 20–29 and 30–39 once each.<br><br><center><img src="images/3940_1.png"></center><br><br>Dr. Grey’s histogram generator is a simple tool. First, the height of the histogram is fixed, that is, the height of the highest bar is always the same and those of the others are automatically adjusted proportionately. Second, the widths of bars are also fixed. It can only produce a histogram of uniform intervals, that is, each interval of a histogram should have the same width (10 in the above example). Finally, the bar for each interval is painted in a grey color, where the colors of the leftmost and the rightmost intervals are black and white, respectively, and the darkness of bars monotonically decreases at the same rate from left to right. For instance, in Figure 1, the darkness levels of the four bars are 1, 2=3, 1=3, and 0, respectively.<br>In this problem, you are requested to estimate ink consumption when printing a histogram on paper. The amount of ink necessary to draw a bar is proportional to both its area and darkness.<br></div><p class="pst">Input</p><div class="ptx" lang="en-US">The input consists of multiple datasets, each of which contains integers and specifies a value table and intervals for the histogram generator, in the following format.<br><p style="padding-left: 30px"><br>n w<br>v1<br>v2<br>.<br>.<br>.<br>vn<br></p><br>n is the total number of value occurrences for the histogram, and each of the n lines following the first line contains a single value. Note that the same value may possibly occur multiple times.<br>w is the interval width. A value v is in the first (i.e. leftmost) interval if 0 ≤ v < w, the second one if w ≤ v < 2w, and so on. Note that the interval from 0 (inclusive) to w (exclusive) should be regarded as the leftmost even if no values occur in this interval. The last (i.e. rightmost) interval is the one that includes the largest value in the dataset.<br>You may assume the following.<br><center><br>1 ≤ n ≤ 100<br>10 ≤ w ≤ 50<br>0 ≤ vi ≤ 100 for 1 ≤ i ≤ n<br></center><br>You can also assume that the maximum value is no less than w. This means that the histogram has more than one interval.<br>The end of the input is indicated by a line containing two zeros.</div><p class="pst">Output</p><div class="ptx" lang="en-US">For each dataset, output a line containing the amount of ink consumed in printing the histogram.<br>One unit of ink is necessary to paint one highest bar black. Assume that 0:01 units of ink per<br>histogram is consumed for various purposes except for painting bars such as drawing lines and<br>characters (see Figure 1). For instance, the amount of ink consumed in printing the histogram<br>in Figure 1 is:<br><center><img src="formula?tex=1%5Ctimes1%2B%5Cfrac%7B2%7D%7B3%7D%5Ctimes%5Cfrac%7B3%7D%7B5%7D%2B%5Cfrac%7B1%7D%7B3%7D%5Ctimes%5Cfrac%7B1%7D%7B5%7D%2B0%5Ctimes%5Cfrac%7B1%7D%7B5%7D%2B0.01+%3D+1%2B%5Cfrac%7B2%7D%7B5%7D%2B%5Cfrac%7B1%7D%7B15%7D%2B0.01+%3D+1.47666..." alt="1\times1+\frac{2}{3}\times\frac{3}{5}+\frac{1}{3}\times\frac{1}{5}+0\times\frac{1}{5}+0.01 = 1+\frac{2}{5}+\frac{1}{15}+0.01 = 1.47666..." align="absmiddle"></center><br>Each output value should be in a decimal fraction and may have an error less than 10<sup>-5</sup>.</div><p class="pst">Sample Input</p><pre class="sio">3 50
