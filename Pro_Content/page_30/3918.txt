<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 31</td><td width="10px"></td><td><b>Accepted:</b> 13</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Roman numerals use symbols I, V, X, L, C, D, and M with values 1, 5, 10, 50, 100, 500, and 1000 respectively. There is an easy evaluation rule for them:<br>Rule ∆: Add together the values for each symbol that is either the rightmost or has a symbol of no greater value directly to its right. Subtract the values of all the other symbols.<br>For example: MMCDLXIX = 1000 + 1000 - 100 + 500 + 50 + 10 - 1 + 10 = 2469.<br>Further rules are needed to uniquely specify a Roman numeral corresponding to a positive integer less than 4000:<br>1. The numeral has as few characters as possible. (IV not IIII)<br>2. All the symbols that make positive contributions form a non-increasing subsequence. (XIV, not VIX)<br>3. All subtracted symbols appear as far to the right as possible. (MMCDLXIX not MCMDLIXX)<br>4. Subtracted symbols are always for a power of 10, and always appear directly to the left of a symbol 5 or 10 times as large that is added. No subtracted symbol can appear more than once in a numeral.<br>Rule 4 can be removed to allow shorter numerals, and still use the same evaluation rule: IM = -1 + 1000 = 999, ILIL = -1 + 100 + -1 + 100 = 198, IVL = -1 -5 + 100 = 94. This would not make the numerals unique, however. Two choices for 297 would be CCVCII and ICICIC. To eliminate the second choice in this example, Rule 4 can be replaced by<br>4'. With a choice of numeral representations of the same length, use one with the fewest subtracted symbols.<br>Finally, replace the Roman numeral symbols to make a system that is more regular and allows larger numbers: Assign the English letter symbols a, A, b, B, c, C, …, y, Y, z, and Z to values 1, 5, 10, 5(10), 10<sup>2</sup>, (5)10<sup>2</sup>, ..., 10<sup>24</sup>, (5)10<sup>24</sup>, 10<sup>25</sup>, and (5)10<sup>25</sup> respectively. Though using the whole alphabet makes logical sense, your problem will use only symbols a-R for easier machine calculations. (R = (5)10<sup>17</sup>.)<br>With the new symbols a-Z, the original formation rules 1-3, the alternate rule 4', and the evaluation rule ∆, numerals can be created, called A to Z numerals. Examples: ad = -1 + 1000 = 999; aAc = -1 - 5 + 100 = 94.</div><p class="pst">Input</p><div class="ptx" lang="en-US">The input starts with a sequence of one or more positive integers less than (7)10<sup>17</sup>, one per line. The end of the input is indicated by a line containing only 0.</div><p class="pst">Output</p><div class="ptx" lang="en-US">For each positive integer in the input, output a line containing only an A to Z numeral representing the integer.  <br>Do not choose a solution method whose time is exponential in the number of digits!</div><p class="pst">Sample Input</p><pre class="sio">999

0</pre><p class="pst">Sample Output</p><pre class="sio">ad
