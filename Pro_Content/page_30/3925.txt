<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=3925&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">Minimal Ratio Tree</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 1613</td><td width="10px"></td><td><b>Accepted:</b> 617</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">For a tree, which nodes and edges are all weighted, the ratio of it is calculated according to the following equation.<br><br><center><img src="formula?tex=Ratio%3D%5Cfrac%7B%5Csum%7Bedgeweight%7D%7D%7B%5Csum%7Bnodeweight%7D%7D" alt="Ratio=\frac{\sum{edgeweight}}{\sum{nodeweight}}" align="absmiddle"></center><br><br>Given a complete graph of n nodes with all nodes and edges weighted, your task is to find a tree, which is a sub-graph of the original graph, with m nodes and whose ratio is the smallest among all the trees of m nodes in the graph.</div><p class="pst">Input</p><div class="ptx" lang="en-US">Input contains multiple test cases. The first line of each test case contains two integers n (2&lt;=n&lt;=15) and m (2&lt;=m&lt;=n), which stands for the number of nodes in the graph and the number of nodes in the minimal ratio tree. Two zeros end the input. The next line contains n numbers which stand for the weight of each node. The following n lines contain a diagonally symmetrical n√ón connectivity matrix with each element shows the weight of the edge connecting one node with another. Of course, the diagonal will be all 0, since there is no edge connecting a node with itself. <br> <br>All the weights of both nodes and edges (except for the ones on the diagonal of the matrix) are integers and in the range of [1, 100]. <br> <br>The figure below illustrates the first test case in sample input. Node 1 and Node 3 form the minimal ratio tree. <br><br><center><img src="images/3925_1.png"></center></div><p class="pst">Output</p><div class="ptx" lang="en-US">For each test case output one line contains a sequence of the m nodes which constructs the minimal ratio tree. Nodes should be arranged in ascending order. If there are several such sequences, pick the one which has the smallest node number; if there's a tie, look at the second smallest node number, etc. Please note that the nodes are numbered from 1.</div><p class="pst">Sample Input</p><pre class="sio">3 2 
30 20 10 
0 6 2 
6 0 3 
2 3 0 
2 2 
1 1 
0 2 
2 0 
0 0</pre><p class="pst">Sample Output</p><pre class="sio">1 3 
1 2 
</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Beijing+2008">Beijing 2008</a></div></td></tr></table>
