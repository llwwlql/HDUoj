<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 30000K</td></tr><tr><td><b>Total Submissions:</b> 354</td><td width="10px"></td><td><b>Accepted:</b> 100</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">The following grammar describes a textual notation for a tree with (not necessarily unique) vertex labels: <br>   tree ::= label<br>   tree ::= label ( subtrees )<br>   subtrees ::= tree<br>   subtrees ::= subtrees , tree<br>   label ::= A | B | C | ... | Z<br><br>That is, the representation of a tree consists of a label (which is an uppercase letter) or a label followed by a bracketed ordered list of trees separated by commas. <br>In order to draw such a tree on paper, we must write each label on the page, such that the labels for the subtrees of a vertex are positioned counter-clockwise about the vertex. The labels must be positioned such that non-intersecting line segments connect each vertex to each of its subtrees. That is to say, we draw the normal planar representation of the tree, preserving the order of subtrees. Beyond these constraints, the position, shape, and size of the representation is arbitrary. <br><br>For example, a possible graphical representation for A(B(C,D),E) is <img src=images/1867_1.jpg><br>Given the textual representation for two trees, you are to determine whether or not they are equivalent. That is, do they share a common paper representation? </div><p class="pst">Input</p><div class="ptx" lang="en-US">The first line of input contains t, the number of test cases. Each test case consists of two lines, each specifying a tree in the notation described above. Each line will contain at most 200 characters, and no white space.</div><p class="pst">Output</p><div class="ptx" lang="en-US">For each test case, output a line containing "same" or "different" as appropriate. <br></div><p class="pst">Sample Input</p><pre class="sio">2
A(B(C,D),E)
E(A,B(C,D))
A(B(C,D),E)
E(A(B(C,D)))
</pre><p class="pst">Sample Output</p><pre class="sio">different
same
