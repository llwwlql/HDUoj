<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 2645</td><td width="10px"></td><td><b>Accepted:</b> 466</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">ACM-ICPC judges, sometimes, play computer games, such as Alphariz. In the game of Alphariz, you are given a table (e.g., Table I) of alphabetic characters. At each step of the game, you choose a table entry E containing a non-blank character ch. Then, you recursively identify all the friends of E. The friends are the existing table entries containing the same character ch which are located immediately to the left, right, up, and down directions of E, and their friends in turn, until no new friends can be located. Then, you replace the characters in E and its found friends with blanks. Table II shows the result of applying the latter rule on Table I, starting at entry E in row 1 and column 2, where ch is 'a'.<br><img src=images/2898_1.jpg><br>Then, while still in the same step, you shift all the non-blank characters to the left and the blanks to the right, as is shown in Table III. Next, you shift all non-blank characters down and the blanks are shifted up as is shown in Table IV. Note that possible full blank rows or columns are to be deleted.<br><img src=images/2898_2.jpg><br>You are to write a program that given the initial table and a sequence of selected table entries, apply the above rules for each given table entry, one after another, and report the final resulting table.</div><p class="pst">Input</p><div class="ptx" lang="en-US">The input consists of several test cases. Each test case starts with the character map which is given in m lines of length n characters (1 <= m, n <= 1000). Characters in the initial map are all lowercase letters. After this, there is a single line containing a single integer k which is the number of selected table entries, followed by k lines, containing two integers r (1 <= r <= m) and c (1 <= c <= n) which are the row and the column numbers of the table entry, respectively. Besides, the items are always inside the current table.</div><p class="pst">Output</p><div class="ptx" lang="en-US">The output for the ith test case should start with a line in the following format:<br>Test case #i:<br>After the first line, the final map should be written with the same format as in the input.</div><p class="pst">Sample Input</p><pre class="sio">aabbb
abaab
aaaba
abbaa
1
1 2
aba
bbc
2
1 3
2 2</pre><p class="pst">Sample Output</p><pre class="sio">Test case #1:
bb
bb
bab
bbaa

a
c</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Tehran+2005">Tehran 2005</a></div></td></tr></table>
