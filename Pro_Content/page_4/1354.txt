<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 10000K</td></tr><tr><td><b>Total Submissions:</b> 1520</td><td width="10px"></td><td><b>Accepted:</b> 807</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Assume that there are n (3<=n<=200) boxes identified by A1, A2,..., An , and each box Ai is configured a lock which is different from the others. Now put n keys to the n locks into the n boxes, each box can only hold a key. After locking all the boxes, unclench the boxes named A1, A2 and fetch out the keys in them to unlock the locked boxes. If the two keys can open some box, fetch out the key in the box to unlock other locked boxes again. If we can open all the boxes finally, we call the placement of the n keys good placement. How many are there different good placements of the n keys?</div><p class="pst">Input</p><div class="ptx" lang="en-US">The input file, ending with -1, contains several data, each of which holds a line.</div><p class="pst">Output</p><div class="ptx" lang="en-US">According to every input datum, compute the number of different good placements. Each output data hold two lines, the first line is held by the input datum, followed by a colon, which follows an equal mark before which is an N; the second is held by the number of different good placement of the n keys.</div><p class="pst">Sample Input</p><pre class="sio">6
8
-1
</pre><p class="pst">Sample Output</p><pre class="sio">N=6:
240
N=8:
10080
</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Xi%27an+2002">Xi'an 2002</a></div></td></tr></table>
