<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=3136&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">The Best Name for Your Baby</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 5000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 213</td><td width="10px"></td><td><b>Accepted:</b> 61</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><span lang="en-us"><p>In the year 29XX, the government of a small country somewhere on the earth introduced a law restricting first names of the people only to traditional names in their culture, in order to preserve their cultural uniqueness. The linguists of the country specifies a set of rules once every year, and only names conforming to the rules are allowed in that year. In addition, the law also requires each person to use a name of a specific length calculated from one&rsquo;s birth date because otherwise too many people would use the same very popular names. Since the legislation of that law, the common task of the parents of new babies is to find the name that comes first in the alphabetical order among the legitimate names of the given length because names earlier in the alphabetical order have various benefits in their culture.</p><p>Legitimate names are the strings consisting of only lowercase letters that can be obtained by repeatedly applying the rule set to the initial string &ldquo;S&rdquo;, a string consisting only of a single uppercase S.</p><p>Applying the rule set to a string is to choose one of the rules and apply it to the string. Each of the rules has the form <i>A</i> &rarr; <i>&alpha;</i>, where <i>A</i> is an uppercase letter and <i>&alpha;</i> is a string of lowercase and/or uppercase letters. Applying such a rule to a string is to replace an occurrence of the letter <i>A</i> in the string to the string <i>&alpha;</i>. That is, when the string has the form &ldquo;<i>&beta;A&gamma;</i>&rdquo;, where <i>&beta;</i> and <i>&gamma;</i> are arbitrary (possibly empty) strings of letters, applying the rule rewrites it into the string &ldquo;<i>&beta;&alpha;&gamma;</i>&rdquo;. If there are two or more occurrences of <i>A</i> in the original string, an arbitrary one of them can be chosen for the replacement.</p><p>Below is an example set of rules.</p><blockquote><table border="0" width="200"><tr><td width="15%" align="right">S</td><td width="15%" align="center">&rarr;</td><td width="55%">aAB</td><td width="15%">(1)</td></tr><tr><td width="15%" align="right">A</td><td width="15%" align="center">&rarr;</td><td width="55%"></td><td width="15%">(2)</td></tr><tr><td width="15%" align="right">A</td><td width="15%" align="center">&rarr;</td><td width="55%">Aa</td><td width="15%">(3)</td></tr><tr><td width="15%" align="right">B</td><td width="15%" align="center">&rarr;</td><td width="55%">AbbA</td><td width="15%">(4)</td></tr></table></blockquote><p>Applying the rule (1) to &ldquo;S&rdquo;, &ldquo;aAB&rdquo; is obtained. Applying (2) to it results in &ldquo;aB&rdquo;, as A is replaced by an empty string. Then, the rule (4) can be used to make it &ldquo;aAbbA&rdquo;. Applying (3) to the first occurrence of A makes it &ldquo;aAabbA&rdquo;. Applying the rule (2) to the A at the end results in &ldquo;aAabb&rdquo;. Finally, applying the rule (2) again to the remaining A results in &ldquo;aabb&rdquo;. As no uppercase letter remains in this string, &ldquo;aabb&rdquo; is a legitimate name.</p><p>We denote such a rewriting process as follows.</p><blockquote><p>S&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(1)</sup></span><span style="margin-left:-1.8ex">aAB</span>&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(2)</sup></span><span style="margin-left:-1.8ex">aB</span>&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(4)</sup></span><span style="margin-left:-1.8ex">a</span>AbbA&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(3)</sup></span><span style="margin-left:-1.8ex">aAabbA</span>&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(2)</sup></span><span style="margin-left:-1.8ex">aAa</span>bb&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(2)</sup></span><span style="margin-left:-1.8ex">a</span>abb</p></blockquote><p>Linguists of the country may sometimes define a ridiculous rule set such as follows.</p><blockquote><table border="0" width="170"><tr><td width="20%" align="right">S</td><td width="20%" align="center">&rarr;</td><td width="40%">sA</td><td width="20%">(1)</td></tr><tr><td width="20%" align="right">A</td><td width="20%" align="center">&rarr;</td><td width="40%">aS</td><td width="20%">(2)</td></tr><tr><td width="20%" align="right">B</td><td width="20%" align="center">&rarr;</td><td width="40%">b</td><td width="20%">(3)</td></tr></table></blockquote><p>The only possible rewriting sequence with this rule set is:</p><blockquote><p>S&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(1)</sup></span><span style="margin-left:-1.8ex">sA</span>&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(2)</sup></span><span style="margin-left:-1.8ex">saS</span>&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(1)</sup></span><span style="margin-left:-1.8ex">sasA</span>&rarr;<span style="position:relative;bottom:0.7ex;left:-2.1ex;"><sup>(2)</sup></span><span style="margin-left:-1.8ex;">&hellip;</span></p></blockquote><p>which will never terminate. No legitimate names exist in this case. Also, the rule (3) can never be used, as its left hand side, B, does not appear anywhere else.</p><p>It may happen that no rules are supplied for some uppercase letters appearing in the rewriting steps. In its extreme case, even S might have no rules for it in the set, in which case there are no legitimate names, of course. Poor nameless babies, sigh!</p><p>Now your job is to write a program that finds the name earliest in the alphabetical order among the legitimate names of the given length conforming to the given set of rules.</p></span></div><p class="pst">Input</p><div class="ptx" lang="en-US"><span lang="en-us"><p>The input is a sequence of datasets, followed by a line containing two zeros separated by a space representing the end of the input. Each dataset starts with a line including two integers <i>n</i> and <i>l</i> separated by a space, where <i>n</i> (1 &le; <i>n</i> &le; 50) is the number of rules and <i>l</i> (0 &le; <i>l</i> &le; 20) is the required length of the name. After that line, <i>n</i> lines each representing a rule follow. Each of these lines starts with one of uppercase letters, A to Z, followed by the character &ldquo;=&rdquo; (instead of &ldquo;&rarr;&rdquo;) and then followed by the right hand side of the rule which is a string of letters A to Z and a to z. The length of the string does not exceed 10 and may be zero. There appears no space in the lines representing the rules.</p></span></div><p class="pst">Output</p><div class="ptx" lang="en-US"><span lang="en-us"><p>The output consists of the lines showing the answer to each dataset in the same order as the input. Each line is a string of lowercase letters, a to z, which is the first legitimate name conforming to the rules and the length given in the corresponding input dataset. When the given set of rules has no conforming string of the given length, the corresponding line in the output should show a single hyphen, &ldquo;<code>-</code>&rdquo;. No other characters should be included in the output.</p></span></div><p class="pst">Sample Input</p><pre class="sio">4 3
