<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 6000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 435</td><td width="10px"></td><td><b>Accepted:</b> 103</td></tr><tr><td colspan="3" align="center" style="color:#0ead00"><b>Case Time Limit:</b> 2000MS</td></tr></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><span lang="en-us"><p>In the world of telecommunications phone calls to different phone numbers have to be charged using different rate or different billing plan. International Carrier of Phone Communications (ICPC) has an antique billing table that determines which phone call has to be charged using which billing plan.</p><p>Each international phone number has 11 digits. The billing table has <i>n</i> lines. Each line specifies a range of prefixes of phone numbers like &ldquo;<code>7919 - 921</code>&rdquo;. This specification means that all phone numbers starting from 7919, 7920, and 7921 match this line. A billing plan name is specified for each prefix. To determine a billing plan for a call, the table is scanned from top to bottom and the first matching line determines the billing plan. If no match is found, the phone number is invalid and no billing plan is needed. A special billing plan named &ldquo;<code>invalid</code>&rdquo; (without quotes) is used as an alternative way to define invalid phone numbers. Some billing plans are used for quite differently looking phone numbers and their names may be specified on different lines in different places of the table.</p><p>ICPC&rsquo;s billing table is old and contains many entries. Some of those entries may not be even used anymore. It is very hard to figure out which phone numbers each billing plan is actually used for. The ICPC&rsquo;s management has reached a decision to transform this billing table into a more legible format. In this new format table consists of the lexicographically ordered list of simple prefixes (without the &ldquo;<code>-</code>&rdquo; range feature of the old format) with a billing plan name for each prefix. No prefix of this new billing table should be a prefix of any other prefix from the table. Thus, a simple dictionary lookup (binary search, for example) will be sufficient to figure out a billing plan for a given phone number. Finding all phone numbers for a given billing plan will also become quite a simple task. The number of lines in the new billing table should be minimized. Billing plan named &ldquo;<code>invalid</code>&rdquo; should not be present in the new billing table at all, since invalid phone numbers will be denoted by absence of the corresponding prefix in the new billing table.</p></span></div><p class="pst">Input</p><div class="ptx" lang="en-US"><span lang="en-us"><p>The first line of the input file contains a single integer number n (1 &le; <i>n</i> &le; 100) &mdash; the number of lines in the old billing table. The following n lines describe the old billing table with one rule on a line. Each rule contains four tokens separated by spaces &mdash; prefix <i>A</i>, minus sign (&ldquo;<code>-</code>&rdquo;), prefix <i>B</i>, and billing plan name. Prefixes contain from 1 to 11 digits each, and the billing plan name contains from 1 to 20 lower case letters.</p><p>Further, let us denote with |<i>A</i>| the number of digits in the prefix <i>A</i>. It is true that 1 &le; |<i>B</i>| &le; |<i>A</i>| &le; 11. Moreover, last |<i>B</i>| digits of prefix <i>A</i> form a string that is lexicographically equal or precedes <i>B</i>. Such pair of prefixes <i>A</i> and <i>B</i> matches all phone numbers with the first |<i>A</i>| &minus; |<i>B</i>| digits matching the first digits of <i>A</i> and with the following |<i>B</i>| digits being lexicographically between the last |<i>B</i>| digits of <i>A</i> and <i>B</i> (inclusive).</p></span></div><p class="pst">Output</p><div class="ptx" lang="en-US"><span lang="en-us"><p>Write to the output file a single integer number <i>k</i> &mdash; the minimal number of lines that the new table should contain to describe the given old billing table. Then write <i>k</i> lines with the lexicographically ordered new billing table. Write two tokens separated by a space on each line &mdash; the prefix and the billing plan name. Note, that the prefix in the new billing table shall contain at least one digit.</p><p>If all phone numbers are invalid (every phone number has no matching line or matches line with billing plan &ldquo;<code>invalid</code>&rdquo;) then the output file should contain just number zero.</p></span></div><p class="pst">Sample Input</p><pre class="sio">8
