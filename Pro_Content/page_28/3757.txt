<div style='position: absolute; right: 10px;'>Language:<select size=1 onChange="window.location.href='problem?id=3757&lang='+this.value+'&change=true'"><option value='default' selected>Default</option></select></div><div class="ptt" lang="en-US">Simple Distributed storage system</div>
<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 2083</td><td width="10px"></td><td><b>Accepted:</b> 371</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><div><p>Jack helps his mentor build a simple distributed storage system. He uses several servers and connects them as the following topology: </p><center><img src="images/3757.jpg"></center><p>This distributed storage system contains a group of backend servers and a master server. The backend servers are different from each other while they store identical data, and all of them are invisible to client. When a client machine needs a file, it sends request to a master server. The master server collects different part of the file from some of backend servers. The strategy of the master is like follows. </p><p>Each backend server has its own processing throughput and transmission bandwidth. The master server knows that i<sup>th</sup> backend serverâ€™s throughput is p<sub>i</sub> (MB/s) and bandwidth is b<sub>i</sub> (MB/s). As the result, omitted the propagation time, handling size of f<sub>i</sub> MB data and sending to master machine needs time: </p><center><p>Total time = Processing time + Transmission time = f<sub>i</sub> / p<sub>i</sub> + f<sub>i</sub> / b<sub>i</sub> </p></center><p>In addition, handling 1 MB data on ith server costs c<sub>i</sub>. (Including electricity consumption and maintenance cost, etc.) In order to minimize the total cost, the master server should carefully decide which backends should be used and how much load they should process. At the same time, because of some consistency consideration, every time the master should choose exactly <b>K</b> backend servers to extract file, and each of them should take exactly the same time to finish the job. </p><p>Your task is to write a scheduling program for the master server. Assuming the size of the file is <b>F</b> MB, and the file can be infinitely divided. </p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><div><p>The first line contains two integers <b>N</b> and <b>K</b> (<b>K</b>&le; <b>N</b> &le; 20000), and a real number <b>F</b>. The master should choose exactly K machines among total <b>N</b> backend servers. The <b>F</b> is the size of the file. </p><p>The following <b>N</b> lines describe the details of each backend servers. Each line contains three real numbers, p<sub>i</sub>, b<sub>i</sub> and c<sub>i</sub>, representing the processing throughput, bandwidth and unit cost. </p></div></div><p class="pst">Output</p><div class="ptx" lang="en-US"><div><p>The output file contains only one real number, the minimum cost. The answer is less than 10000000000 and should be rounded to four digits </p></div></div><p class="pst">Sample Input</p><pre class="sio">3 2 2
1 1 2
1 1 1
2 2 10
</pre><p class="pst">Sample Output</p><pre class="sio">3.0000</pre><p class="pst">Hint</p><div class="ptx" lang="en-US">In the sample case, the master should choose the first two backend machines. Each of them should handle 1 MB part of the file (total is 2 MB) in order to make the finishing time identically (2 second). The total cost is 1*2+1*1 = 3.0000</div><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=POJ+Monthly+Contest+-+2010.04.04">POJ Monthly Contest - 2010.04.04</a>, Yao Jinyu</div></td></tr></table>
