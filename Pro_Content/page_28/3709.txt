<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 4000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 5606</td><td width="10px"></td><td><b>Accepted:</b> 1808</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><div><p>The explosively increasing network data in various application domains has raised privacy concerns for the individuals involved. Recent studies show that simply removing the identities of nodes before publishing the graph/social network data does not guarantee privacy. The structure of the graph itself, along with its basic form the degree of nodes, can reveal the identities of individuals. </p><p>To address this issue, we study a specific graph-anonymization problem. We call a graph <i>k</i>-anonymous if for every node <i><b>v</b></i>, there exist at least <i>k</i>-1 other nodes in the graph with the same degree as <i><b>v</b></i>. And we are interested in achieving <i>k</i>-anonymous on a graph with the minimum number of graph-modification operations. </p>We simplify the problem. Pick <i>n</i> nodes out of the entire graph <i><b>G</b></i> and list their degrees in ascending order. We define a sequence <i>k</i>-anonymous if for every element <i><b>s</b></i>, there exist at least <i>k</i>-1 other elements in the sequence equal to <i><b>s</b></i>. To let the given sequence <i>k</i>-anonymous, you could do one operation only—decrease some of the numbers in the sequence. And we define the cost of the modification the sum of the difference of all numbers you modified. e.g. sequence 2, 2, 3, 4, 4, 5, 5, with <i>k</i>=3, can be modified to 2, 2, 2, 4, 4, 4, 4, which satisfy 3-anonymous property and the cost of the modification will be |3-2| + |5-4| + |5-4| = 3. <p>Give a sequence with <i>n</i> numbers in ascending order and <i>k</i>, we want to know the modification with minimal cost among all modifications which adjust the sequence <i>k</i>-anonymous. </p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><div><p>The first line of the input file contains a single integer <i>T</i> (1 &le; <i>T</i> &le; 20) – the number of tests in the input file. Each test starts with a line containing two numbers <i>n</i> (2 &le; <i>n</i> &le; 500000) – the amount of numbers in the sequence and <i>k</i> (2 &le; <i>k</i> &le; <i>n</i>). It is followed by a line with <i>n</i> integer numbers—the degree sequence in ascending order. And every number s in the sequence is in the range [0, 500000]. </p></div></div><p class="pst">Output</p><div class="ptx" lang="en-US"><p>For each test, output one line containing a single integer—the minimal cost.</p></div><p class="pst">Sample Input</p><pre class="sio">2
7 3
2 2 3 4 4 5 5
6 2
0 3 3 4 8 9
</pre><p class="pst">Sample Output</p><pre class="sio">3
5
