<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 2115</td><td width="10px"></td><td><b>Accepted:</b> 659</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><div><p>Harry and Sally were playing games at Christmas Eve. They drew some Christmas trees on a paper: </p><div align="center"><img src="images/3710_1.png"> </div><p>Then they took turns to cut a branch of a tree, and removed the part of the tree which had already not connected with the root. A step shows as follows: </p><div align="center"><img src="images/3710_2.png"> </div><p>Sally always moved first. Who removed the last part of the trees would win the game. </p><p>After a while, they all figured out the best strategy and thought the game was too simple for them. Harry said, “The Christmas trees should have some gifts in them!” So Sally drew some gifts (simple polygons) in the initial trees: </p><div align="center"><img src="images/3710_3.png"> </div><p>You may assume the initial picture is a tree with some simple polygons, in which each edge is involved in at most one polygon. We also know that every polygon has only one node involved in the main tree (the hanging point of the giftJ) .In every sub-tree (connected subgraph), there was one and only one node representing the “root”. According to these assumptions, following graphs will never appear: </p><div align="center"><img src="images/3710_4.png"> </div><p>Sally and Harry took turns (Sally was always the first person to move), to cut an edge in the graph, and removed the part of the tree that no longer connected to the root. The person who cannot make a move lost the game. </p><p>Your job is to decide who will finally win the game if both of them use the best strategy. </p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><div><p>The input file contains multiply test cases.<br>The first line of each test case is an integer <i>N</i> (<i>N</i>&lt;100), which represents the number of sub-trees. The following lines show the structure of the trees. The first line of the description of a tree is the number of the nodes <i>m</i> (<i>m</i>&lt;100) and the number of the edges <i>k</i> (<i>k</i>&lt;500). The nodes of a tree are numbered from 1 to <i>m</i>. Each of following lines contains 2 integers <i>a</i> and <i>b</i> representing an edge &lt;<i>a</i>, <i>b</i>&gt. Node 1 is always the root. </p></div></div><p class="pst">Output</p><div class="ptx" lang="en-US"><div><p>For each test case, output the name of the winner. </p></div></div><p class="pst">Sample Input</p><pre class="sio">2
2 1
1 2
4 4
1 2
2 3
2 4
3 4
</pre><p class="pst">Sample Output</p><pre class="sio">Sally</pre><p class="pst">Hint</p><div class="ptx" lang="en-US"><div><p>The sample graph is</p><img src="images/3710_5.png"></div><br></div><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=POJ+Founder+Monthly+Contest+%E2%80%93+2008.12.28">POJ Founder Monthly Contest – 2008.12.28</a>, Yaojinyu</div></td></tr></table>
