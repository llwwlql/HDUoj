<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 2734</td><td width="10px"></td><td><b>Accepted:</b> 932</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><div><p><i>Bubble sort is a simple sorting algorithm. It works by repeatedly stepping through the list to be sorted, comparing each pair of adjacent items and swapping them if they are in the wrong order. The pass through the list is repeated until no swaps are needed, which indicates that the list is sorted. The algorithm gets its name from the way smaller elements "bubble" to the top of the list. Because it only uses comparisons to operate on elements, it is a comparison sort.<br />                ­­­­­­­­­­ Wikipedia</i></p><p>Bubble Sort is a very simple sorting algorithm which runs in O(n<sup>2</sup>) time. Each round, we start from the beginning of the list, compare each adjacent pair of items in turn, swapping the items if necessary. Repeat the pass through the list, until no swaps are done. Assume that after exactly T rounds, the array is already in the ascending order, then we say that T is the number of Bubble Sort Rounds of this array. Below is an example: Let us take an array of numbers "5 1 4 2 8", then we sort the array using Bubble Sort as follow:</p><p>First Round:<br />( <b>5 1</b> 4 2 8 ) -> ( <b>1 5</b> 4 2 8 ), Compares the first two elements, and swaps them.<br />( 1 <b>5 4</b> 2 8 ) -> ( 1 <b>4 5</b> 2 8 ), Swap since 5 > 4<br />( 1 4 <b>5 2</b> 8 ) -> ( 1 4 <b>2 5</b> 8 ), Swap since 5 > 2<br />( 1 4 2 <b>5 8</b> ) -> ( 1 4 2 <b>5 8</b> ), since these elements are already in order (8 > 5), algorithm does not swap them.<br />Second Round:<br />( <b>1 4</b> 2 5 8 ) -> ( <b>1 4</b> 2 5 8 )<br />( 1 <b>4 2</b> 5 8 ) -> ( 1 <b>2 4</b> 5 8 ), Swap since 4 > 2<br />( 1 2 <b>4 5</b> 8 ) -> ( 1 2 <b>4 5</b> 8 )<br />( 1 2 4 <b>5 8</b> ) -> ( 1 2 4 <b>5 8</b> )</p><p>After T = 2 rounds, the array is already sorted, hence we say that the number of Bubble Sort Rounds of this array is equal to 2.</p><p>ZX learns Bubble Sort in an algorithm class and his teacher leaves him a problem as homework. The teacher gives ZX an array <i>A</i> with <i>N</i> distinct numbers which is already sorted in ascending order and he tells ZX that this array is obtained after exactly <i>K</i> rounds of Bubble sort. The problem is: How many initial arrays there may be from which we can obtain the array <i>A</i> after exactly <i>K</i> rounds of Bubble Sort? The result may be very large, so you only need to output the answer mod 20100713. 　</p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><div>The input may contain several cases.<br />The first line contains an integer <i>T</i> (<i>T</i> ≤ 100,000), indicating the number of test cases.<br />Then <i>T</i> lines of test cases follow. For each line, it contains two integers <i>N</i> and <i>K</i> (1 ≤ <i>N</i> ≤ 1,000,000, 0 ≤ <i>K</i> ≤ <i>N</i> - 1) where <i>N</i> is the size of array and <i>K</i> is the number of Bubble Sort Rounds. </div></div><p class="pst">Output</p><div class="ptx" lang="en-US"><div><p>For each line, output an integer which is the number of initial arrays mod 20100713. </p></div></div><p class="pst">Sample Input</p><pre class="sio">3
3 0
3 1
3 2
</pre><p class="pst">Sample Output</p><pre class="sio">1
3
2
</pre><p class="pst">Hint</p><div class="ptx" lang="en-US"><div><p>Suppose the ordered array is {a, b, c} (a < b < c). For the 6 possible initial arrays:<br />{a, b, c}, {a, c, b}, {b, a, c}, {b, c, a}, {c, a, b}, {c, b, a},<br />we can get that:<br />{a, b, c}: already sorted, no bubble sort needed.<br />{a, c, b}, {b, a, c}, {c, a, b}: we need 1 round of Bubble Sort.<br />{b, c, a}, {c, b, a}: we need 2 rounds of Bubble Sort.<br />　</p></div></div><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key="></a></div></td></tr></table>
