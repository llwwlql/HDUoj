<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 1077</td><td width="10px"></td><td><b>Accepted:</b> 166</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><div><p>While solving non-linear equations in numerical analysis lesson, professor Busoniya introduce us Newton's Method. Like the other ways (such as bisection, Muller's Method), this method is also based on a linear approximation of the function, but does so using a tangent to the curve. The figure below gives a graphical description.</p><center><img src="images/3727.jpg"></center><br> <p>Starting from a single initial estimate, <i>x<sub>0</sub></i>, which is not too far from a root, we move along the tangent to its intersection with the x-axis, and take that as the next approximation <i>x<sub>1</sub></i>. The general term is: <i>x<sub>n+1</sub></i> = <i>x<sub>n</sub></i> - f(<i>x<sub>n</sub></i>) / f′(<i>x<sub>n</sub></i>), n = 0, 1, 2, .... Newton’s algorithm is widely used because it is more rapidly convergent than any of the methods discussed so far. Then now, you task is to accurate how many iterations it takes to get a root using Newton's Method. The <i>x<sub>0</sub></i> and the equation are given and all the tolerance is 0.000001. You just use this method to get the answer. The iteration will stop when |<i>x<sub>n+1</sub></i> - <i>x<sub>n</sub></i>| < tolerance or |f(<i>x<sub>n</sub></i>)| < tolerance then n is the answer. But sometimes the <i>x<sub>0</sub></i> or equation we choose is so bad that we cannot get the answer using this method within 1000 iterations, then just output "Bad x0 or bad equation!" </p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><div><p>There are multiple cases ended by EOF. Each case has two lines, first line is the equation and the second line is <i>x<sub>0</sub></i>. We ensure that each equation just contain integers, 'x', '+', '-', '=' and '^', the right part of '=' is always 0, and there is a space between each term. The number of terms in the equation is less than 50 and all integers in the equations are less than 1000. </p></div></div><p class="pst">Output</p><div class="ptx" lang="en-US"><div><p>For each test case, output n or "Bad x0 or bad equation!" per line. </p></div></div><p class="pst">Sample Input</p><pre class="sio">-3x^2 -3 = 0
3
3x -3 = 0
1
5x^2 -2 = 0
-5
</pre><p class="pst">Sample Output</p><pre class="sio">Bad x0 or bad equation!
0
6
</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=POJ+Monthly+Contest+%E2%80%93+2009.04.05">POJ Monthly Contest – 2009.04.05</a>, longpo</div></td></tr></table>
