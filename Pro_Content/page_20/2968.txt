<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td colspan="3"><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 575</td><td width="10px"></td><td><b>Accepted:</b> 85</td><td width="10px"></td><td style="font-weight:bold; color:red;">Special Judge</td></tr></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><span lang="en-us"><p>There are <b>N</b> points marked on a surface, pair (<b>x<sub>i</sub></b>, <b>y<sub>i</sub></b>) is coordinates of a point number <b>i</b>. Let&rsquo;s call a <b>necklace</b> a set of <b>N</b> figures which fulfills the following rules.</p><ul><li>The figure #<b>i</b> consists of all such points (<b>x</b>, <b>y</b>) that (<b>x</b> &minus; <b>x<sub>i</sub></b>)<sup>2</sup> + (<b>y</b> &minus; <b>y<sub>i</sub></b>)<sup>2</sup> &le; <b>r<sub>i</sub></b><sup>2</sup>, where <b>r<sub>i</sub></b> &ge; 0.</li><li>Figures #<b>i</b> and #(<b>i</b> + <b>1</b>) intersect (1 &le; <b>i</b> &lt; <b>N</b>)</li><li>Figures #<b>1</b> and #<b>N</b> intersect</li><li>All the rest pairs of figures do not intersect.</li></ul><p>Write a program which takes points and constructs a necklace.</p></span></div><p class="pst">Input</p><div class="ptx" lang="en-US"><span lang="en-us"><p>The first line of the input contains one integer number <b>N</b> (2 &le; <b>N</b> &le; 100). Each of the next <b>N</b> lines contains two real numbers <b>x<sub>i</sub></b>, <b>y<sub>i</sub></b> (1 000 &le; <b>x<sub>i</sub></b>, <b>y<sub>i</sub></b> &le; 1 000), separated by one space. It is guaranteed that at least one necklace exists.</p></span></div><p class="pst">Output</p><div class="ptx" lang="en-US"><span lang="en-us"><p>The output has to contain <b>N</b> lines. A line #<b>i</b> contains real number <b>r<sub>i</sub></b>, (0 &le; <b>r<sub>i</sub></b> &lt; 10 000). To avoid potential accuracy problems, a checking program uses the following rules.</p><ul><li>Figures #<b>i</b> and #<b>j</b> definitely do not intersect if (<b>r<sub>i</sub></b> + <b>r<sub>j</sub></b>) &le; <b>d<sub>ij</sub></b> &minus; 10<sup>&minus;5</sup></li><li>Figures #<b>i</b> and #<b>j</b> definitely intersect if (<b>d<sub>ij</sub></b> + 10<sup>&minus;5</sup>) &le; (ri + rj).</li><li>The case when <b>d<sub>ij</sub></b> &minus; 10<sup>&minus;5</sup> &lt; (<b>r<sub>i</sub></b> + <b>r<sub>j</sub></b>) &lt; <b>d<sub>ij</sub></b> + 10<sup>&minus;5</sup> is decided in favour of a contestant.</li><li><b>d<sub>ij</sub></b> equals <b>sqrt((x<sub>i</sub> &minus; x<sub>j</sub>)<sup>2</sup> + (y<sub>i</sub> &minus; y<sub>j</sub>)<sup>2</sup>)</b> in the rules above.</li></ul></span></div><p class="pst">Sample Input</p><pre class="sio">4
0 0
10 0
10 10
0 10</pre><p class="pst">Sample Output</p><pre class="sio">7
7
7
7</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Northeastern+Europe+2004">Northeastern Europe 2004</a>, Western Subregion</div></td></tr></table>
