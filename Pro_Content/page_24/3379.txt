<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 360</td><td width="10px"></td><td><b>Accepted:</b> 110</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US"><p>Johnny is a little boy, and now he is learning alphabet. His father made him a birthday present &#8212; a large collection of tokens, each marked with a letter from &#8216;<tt>A</tt>&#8217; to &#8216;<tt>Z</tt>&#8217;, and Johnny invented an interesting game to help himself learning.</p><p>First, Johnny takes several tokens and puts them in a circle. After that he selects one of the tokens to start the game at, and chooses some number k. Each turn Johnny counts k tokens along the circle to find the next winning token and inserts another token to the circle immediately after it. This new token must hold the letter which follows alphabetically the letter written on the winning token, i.e. &#8216;<tt>B</tt>&#8217; is inserted if the winning token is &#8216;<tt>A</tt>&#8217;, &#8216;<tt>C</tt>&#8217; is inserted after &#8216;<tt>B</tt>&#8217;, etc. If the winning token is &#8216;<tt>Z</tt>&#8217;, the token &#8216;<tt>A</tt>&#8217; is inserted. There are so many tokens available that Johnny can always find one with the required letter.</p><p>After inserting the token Johnny proceeds to another game turn, this time starting from the newly inserted token. The number <i>k</i> is only selected once and is not changed between turns. The first four turns of a sample game with initial tokens &#8216;<tt>J</tt>&#8217;, &#8216;<tt>O</tt>&#8217;, &#8216;<tt>H</tt>&#8217;, &#8216;<tt>N</tt>&#8217;, &#8216;<tt>N</tt>&#8217;, &#8216;<tt>Y</tt>&#8217; (&#8216;<tt>J</tt>&#8217; is the starting one) and <i>k</i> = 3 are shown on a figure below:</p><div align="center"><img src="images/3379_1.png"></div><p>Georgie is Johnny&#8217;s elder brother. Since he is already a schoolboy, he plays the same game mentally and astonishes Johnny with his ability to predict the letter inserted at turn <i>i</i>. But as Johnny gets smarter and makes more and more turns, it becomes harder for Georgie to compete, so he decides to write a computer program to find the results quickly.</p></div><p class="pst">Input</p><div class="ptx" lang="en-US"><p>The first line of the input file contains three integers: <i>n</i> &#8212; the number of tokens initially placed in a circle (1 &#8804; <i>n</i> &#8804; <nobr>10 000</nobr>), <i>k</i> &#8212; the number of tokens counted each turn (1 &#8804; <i>k</i> &#8804; <nobr>10 000</nobr>), and <i>m</i> &#8212; the number of turns (1 &#8804; <i>m</i> &#8804; 10<sup>9</sup>).</p><p>The second line contains a string of <i>n</i> uppercase letters (&#8216;<tt>A</tt>&#8217; to &#8216;<tt>Z</tt>&#8217;) &#8212; the tokens initially placed in a circle, starting from the initial token.</p></div><p class="pst">Output</p><div class="ptx" lang="en-US"><p>Output a single uppercase English letter which is added to the circle at the <i>m</i>-th turn.</p></div><p class="pst">Sample Input</p><pre class="sio">6 3 4
JOHNNY</pre><p class="pst">Sample Output</p><pre class="sio">Z</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Northeastern+Europe+2006">Northeastern Europe 2006</a>, Northern Subregion</div></td></tr></table>
