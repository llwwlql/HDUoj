<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 1000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 65536K</td></tr><tr><td><b>Total Submissions:</b> 1559</td><td width="10px"></td><td><b>Accepted:</b> 573</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Once upon a time in the silent depths of digital forests there lived a Binary Witch. She was able to forecast weather, telling for any day in the future whether it will be rainy or sunny.<br><br>Her magic was based on the following ancient rule: let a1, a2, ..., aN be the sequence of binary digits, where ai = 0 indicates that i-th day was rainy, and ai = 1 -- that it was sunny. To predict the weather in day N+1, consider the t-postfix a<sub>N-t+1</sub>, a<sub>N-t+2</sub>, ..., aN consisting of the last t elements. If that postfix is encountered somewhere before the position N-t+1, i.e. if there is such k <= N-t, that ak = a<sub>N-t+1</sub>, a<sub>k+1</sub> = a<sub>N-t+2</sub>, ..., a<sub>k+t-1</sub> = aN then the predicted value will be a<sub>k+t</sub>.<br><br>If there is more than one occurrence of t-postfix, then the rightmost one (with maximal k) will be taken. So, to make a prediction, she tried t-postfixes, consequently for t = 13, 12, ..., 1, stopping after the first prediction. If neither postfix was found, she predicted rain ("0"). If prediction for more than one day is needed, it is assumed that all previous days are predicted correctly, so if first predicted value is b, then we make forecast for day N+2 based on N+1 values, where a<sub>N+1</sub> = b.<br><br>Because the witch was burned long ago, your task is to write a program to perform her arcane job.</div><p class="pst">Input</p><div class="ptx" lang="en-US">First line of input file contains two integers N (1 <= N <= 1000000) and L (1 <= L <= 1000), separated by space. Second line contains a string of N characters "0" and "1".</div><p class="pst">Output</p><div class="ptx" lang="en-US">Output file must contain a single string of L characters, which are forecasts for days N+1, N+2, ..., N+L.</div><p class="pst">Sample Input</p><pre class="sio">10 7
1101110010</pre><p class="pst">Sample Output</p><pre class="sio">0100100</pre><p class="pst">Source</p><div class="ptx" lang="en-US"><a href="searchproblem?field=source&key=Northeastern+Europe+2000">Northeastern Europe 2000</a>, Far-Eastern Subregion</div></td></tr></table>
