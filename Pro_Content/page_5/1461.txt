<div class="plm"><table align="center"><tr><td><b>Time Limit:</b> 2000MS</td><td width="10px"></td><td><b>Memory Limit:</b> 10000K</td></tr><tr><td><b>Total Submissions:</b> 439</td><td width="10px"></td><td><b>Accepted:</b> 186</td></table></div><p class="pst">Description</p><div class="ptx" lang="en-US">Each mapping f of the set {0,1}<sup>n</sup> of n-dimensional binary vectors to {0,1} is called Boolean function of n variables and denoted by f(x<sub>n</sub>,x<sub>n-1</sub>,...,x1). For cryptography some properties of the Boolean functions are interesting. Let denote by B(n,k) the set of n-dimensional binary vectors that have k 1's. The task is for given Boolean function f to find the number of vectors (b<sub>n</sub>,b<sub>n-1</sub>,...,b1) from B(n,k) such that f(b<sub>n</sub>,b<sub>n-1</sub>,...,b1)=1.  <br><br>The Boolean function will be given by its (unique) polynomial modulo 2. In these polynomials the operations addition and multiplication modulo 2 are used, defined as shown in the tables of Fig. 1. In the polynomial of a function any product of m variables  could participate or not participate. So the general form of the polynomial for n variables is:<br><br>	a0+a1x1+a2x2+a3x2x1+a4x3+a5x3x1+a6x3x2+a7x3x2x1+. . .+aNx<sub>n</sub>x<sub>n-1</sub>...x1<br><br>where all coefficients aj, j=0,1,...,N=2^n-1, are 0's or 1's and if the coefficient is equal to 0 we will omit the corresponding product and if it is equal to 1 we just will omit the coefficient. For example, the polynomial of the Boolean function disjunction of 2 variables given on Fig. 2 is 0+1.x1+1.x2+1.x2x1=x1+x2+x2x1.<br><table align=center width=80%><tr><td><table border=1><tr align=center><td width=30>+</td><td width=30>0</td><td width=30>1</td></tr><tr align=center><td>0</td><td>0</td><td>1</td></tr><tr align=center><td>1</td><td>1</td><td>0</td></tr></table></td><td><table border=1><tr  align=center><td width=30>*</td><td width=30>0</td><td width=30>1</td></tr><tr align=center><td>0</td><td>0</td><td>0</td></tr><tr align=center><td>1</td><td>0</td><td>1</td></tr></table></td><td><table border=1><tr  align=center><td width=30>x1</td><td width=30>x2</td><td width=30>f</td></tr><tr align=center><td>0</td><td>0</td><td>0</td></tr><tr align=center><td>0</td><td>1</td><td>1</td></tr><tr align=center><td>1</td><td>0</td><td>1</td><tr align=center><td>1</td><td>1</td><td>1</td></table></td></tr></table><br><pre>                                    fig.1                                                     fig.2</pre><br></div><p class="pst">Input</p><div class="ptx" lang="en-US">Your program has to be ready to solve more than one test case. The first line of the input will contains only the number T of the test cases. Each of the following T lines will describe one function - first the numbers n and k separated by single space (1 <= n <= 18,0 <= k <= n) and then, separated by one more space a string of 2^n 0's and 1's that are the coefficients of the corresponding polynomial, ordered as in the general form of the polynomial given above.</div><p class="pst">Output</p><div class="ptx" lang="en-US">The output have to contain T lines with a single number each - the number of vectors found by your program. </div><p class="pst">Sample Input</p><pre class="sio">3
